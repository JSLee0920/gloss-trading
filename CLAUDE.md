# Gloss

Gloss is an AI trading/investment **tutor and analyst**, delivered as a companion
app. It teaches and explains markets and analyzes instruments for the user — it is
an educational companion, not a broker or a personalized advice engine.

## Domain guardrails (read before writing product logic or copy)

- **Educational framing, not advice.** Output should teach and analyze. Avoid
  phrasing that reads as personalized financial advice ("you should buy X").
  Prefer "here's how to think about X" / "the data shows Y."
- **Surface risk and uncertainty.** Never present a prediction as a certainty.
  Include the reasoning and the caveats, not just a verdict.
- **No fabricated market data.** Prices, fundamentals, and figures come from a
  real data source. Never hardcode or invent numbers in product code or copy.
- When in doubt about a compliance-sensitive decision, flag it for the user
  rather than deciding silently.

## Stack

- **Next.js 16** (App Router) · **React 19** · **TypeScript** (strict)
- **Tailwind CSS v4** (config-less, via `@tailwindcss/postcss`)
- **React Compiler** enabled (`babel-plugin-react-compiler`)
- **pnpm** package manager
- App lives in **`web/`** — run all commands from there.

## Commands (run from `web/`)

```bash
pnpm dev            # dev server (http://localhost:3000)
pnpm build          # production build
pnpm lint           # eslint
pnpm typecheck      # tsc --noEmit  (see note below)
pnpm test           # vitest unit/component tests (run once)
pnpm test:watch     # vitest watch mode
pnpm e2e            # playwright end-to-end tests
```

## Conventions

- **Server Components by default.** Only add `"use client"` when a file needs
  browser state, effects, or event handlers. Keep the client boundary as low in
  the tree as possible.
- **Do not manually memoize.** React Compiler handles it — no `useMemo`,
  `useCallback`, or `React.memo` unless profiling proves a specific need.
- **Imports:** use the `@/*` alias for `src/*` (e.g. `@/components/Button`), not
  long relative paths.
- **Routing:** App Router under `src/app`. Group routes with `(group)` folders
  (e.g. `(auth)/login`) — parentheses groups don't affect the URL.
- **Styling:** Tailwind utilities inline. Use semantic tokens
  (`text-muted-foreground`, `bg-foreground`, `text-background`) over raw colors
  so light/dark and theming stay consistent.
- **Metadata:** every route exports a `metadata` object with a real `title` and
  `description` (the placeholder "Generated by create next app" is a bug — fix it
  when you touch a file that still has it).
- **Accessibility:** label every input (`htmlFor`/`id`), keep semantic HTML
  (`main`, `h1`, `form`).

## Testing

- **Unit/component:** Vitest + Testing Library, jsdom environment. Co-locate as
  `*.test.tsx` next to the component.
- **E2E:** Playwright specs in `web/e2e/*.spec.ts`. Keep at least one smoke test
  green (home → `/login` redirect).
- After any non-trivial change, run `pnpm typecheck && pnpm test`. Don't call a
  change done until both pass.
